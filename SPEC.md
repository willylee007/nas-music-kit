# NAS 音乐助手 Spec

## 1. 概述

NAS 音乐助手是一个面向个人 NAS 音乐库入库场景的 Web 工具。用户可以通过浏览器搜索音乐、试听、查看音质与文件大小、下载歌曲或歌词，并将文件保存到服务器指定的音乐目录中。

项目由 Flask 后端和单页 HTML 前端组成，支持普通模式与 VIP 模式入口，并提供 PWA 安装能力。

## 2. 目标

- 提供简洁的音乐搜索与下载体验。
- 支持多音源搜索、试听、歌词和高音质下载。
- 支持将下载结果直接保存到 NAS 挂载目录。
- 支持批量选择与批量下载，减少重复操作。
- 支持常用选项本地缓存，提升重复使用效率。
- 支持通过分享链接快速解析并下载歌曲。

## 3. 非目标

- 不提供用户账号体系。
- 不提供云端同步或多用户权限管理。
- 不提供音乐库管理、刮削、去重或播放列表管理。
- 不缓存第三方音乐资源。
- 不对第三方接口的可用性做强保证。

## 4. 用户场景

### 4.1 搜索单曲

用户输入歌手或歌名，选择音源和音质，点击搜索，系统返回歌曲列表。用户可以试听、查看文件信息、下载歌曲或歌词。

### 4.2 搜索专辑

用户将搜索类型切换为“专辑”，输入专辑名称，系统按音源接口要求使用专辑搜索参数，返回专辑内曲目列表。

### 4.3 批量下载

用户在搜索结果中勾选多首歌曲，点击“下载选中”，系统依次下载选中歌曲并显示每首歌曲的下载进度。

### 4.4 自动保存目录

用户选择保存目录规则：

- 无：保存到下载根目录。
- 歌手：保存到歌手名子目录。
- 专辑：保存到专辑名子目录。

### 4.5 分享链接快速下载

用户粘贴支持平台的歌曲分享链接，系统解析链接中的歌曲 ID，补全元数据并直接下载。

## 5. 功能需求

### 5.1 页面入口

- `/`：普通模式入口。
- `/vip`：VIP 模式入口。
- 两个入口复用同一前端模板，通过后端注入 `vip_mode` 区分能力范围。

### 5.2 搜索区

搜索区包含：

- 搜索类型下拉框，位于搜索输入框内：
  - 歌手/歌曲
  - 专辑
- 搜索输入框：
  - 歌手/歌曲模式 placeholder 为“请输入歌手、歌名”。
  - 专辑模式 placeholder 为“请输入专辑名称”。
- 搜索按钮。
- 音源下拉框。
- 音质下拉框。
- 保存目录下拉框：
  - 无
  - 歌手
  - 专辑

### 5.3 本地选项缓存

前端使用 `localStorage` 缓存以下内容：

- 搜索类型。
- 搜索输入内容。
- 音源。
- 音质。
- 保存目录规则。

页面加载时应优先恢复缓存值。缓存中的下拉值必须存在于当前页面选项中才恢复，避免普通模式和 VIP 模式选项不一致导致无效值。

### 5.4 音源支持

普通模式支持：

- 网易云音乐 1
- 网易云音乐 2
- 酷我音乐
- JOOX
- B 站

VIP 模式支持：

- 网易云音乐
- 网易云音乐 2
- QQ 音乐
- 酷我音乐
- Tidal
- Qobuz
- JOOX
- B 站
- Apple Music
- YouTube Music
- Spotify

### 5.5 搜索逻辑

#### 5.5.1 歌手/歌曲搜索

前端请求：

```text
GET /api/search?source={source}&name={keyword}&pages={page}&search_type=song
```

后端对 GD Studio 接口请求：

```text
GET https://music-api.gdstudio.xyz/api.php?types=search&source={source}&name={keyword}&count={count}&pages={page}
```

#### 5.5.2 专辑搜索

前端请求：

```text
GET /api/search?source={source}&name={keyword}&pages={page}&search_type=album
```

后端对 GD Studio 接口请求时，应将 `source` 转换为 `{source}_album`：

```text
GET https://music-api.gdstudio.xyz/api.php?types=search&source={source}_album&name={keyword}&count={count}&pages={page}
```

后端返回给前端的结果中，`source` 应还原为可用于试听、封面、歌词和下载的原始音源值。

#### 5.5.3 网易云音乐 2 兼容

网易云音乐 2 使用 BugPk 专线处理单曲搜索和下载。专辑搜索不走 BugPk，选择网易云音乐 2 且搜索类型为专辑时，后端应使用网易云音乐 1 的 GD Studio 专辑搜索能力。

### 5.6 搜索结果展示

每条搜索结果展示：

- 选择框。
- 封面图。
- 歌曲名。
- 歌手。
- 专辑。
- 音质/文件大小查询按钮。
- 试听按钮。
- 仅下载歌词按钮。
- 下载歌曲按钮。

交互要求：

- 点击歌手名，以“歌手/歌曲”模式搜索该歌手。
- 点击专辑名，以“专辑”模式搜索该专辑。
- 点击卡片空白区域可切换选中状态。
- 点击按钮、选择框、歌手或专辑链接不应触发卡片选中切换。

### 5.7 分页

- 搜索结果使用上一页/下一页翻页。
- 当前页小于等于 1 时禁用上一页。
- 返回结果少于每页数量时禁用下一页。
- 翻页或新搜索时应停止当前试听并清空已选歌曲。

### 5.8 试听

- 用户点击试听按钮后，前端请求 `/api/preview` 获取试听链接。
- 同一首歌重复点击可播放/暂停。
- 切换到另一首歌时应停止当前播放。
- 播放中的卡片显示进度条。

### 5.9 音质与文件大小查询

- 用户点击信息按钮后，前端请求 `/api/info`。
- 成功时显示音质等级和文件大小徽标。
- 已加载过信息的卡片不重复请求。

### 5.10 下载歌曲

- 单曲下载通过 `/api/download/start` 创建后台下载任务。
- 前端轮询 `/api/download/progress/{job_id}` 获取进度。
- 下载完成后按钮显示成功状态。
- 下载失败后按钮显示失败状态并允许重试。

下载流程应包括：

1. 获取音乐资源 URL。
2. 获取封面。
3. 下载音频文件。
4. 根据响应类型或 URL 判断扩展名。
5. 写入标题、歌手、专辑、封面等音频标签。
6. 可选写入歌词文件。

### 5.11 仅下载歌词

- 用户点击歌词按钮后，前端请求 `/api/lyric`。
- 后端获取歌词并保存为 `.lrc` 文件。
- 文件名规则与歌曲下载保持一致：`歌手 - 歌名.lrc`。

### 5.12 批量下载

- 用户可全选、清空选择、下载选中歌曲。
- 批量下载按顺序逐首执行。
- 批量下载过程中禁用批量下载按钮。
- 完成后显示成功和失败数量。

### 5.13 保存目录规则

前端根据保存目录选项生成 `subdir`：

- 无：传空字符串。
- 歌手：传当前歌曲歌手名。
- 专辑：传当前歌曲专辑名。

目录名应在前端移除不适合作为路径名的字符：

```text
\ / : * ? " < > |
```

后端仍必须进行安全校验：

- 不允许绝对路径。
- 不允许空字符。
- 不允许 `..` 路径片段。
- 最终目录必须位于下载根目录内。

### 5.14 分享链接解析

支持从文本中提取 URL，并解析以下平台链接：

- 网易云音乐：`music.163.com`、`163cn.tv`。
- QQ 音乐：`y.qq.com`。
- 酷我音乐：`kuwo.cn`。

短链应先还原真实 URL。

快速下载流程：

1. 提取链接。
2. 解析平台和歌曲 ID。
3. 补全元数据。
4. 使用内部下载核心流程下载。

当前快速链接下载不根据前端保存目录规则生成歌手或专辑目录。

### 5.15 PWA

- 提供 `manifest.json`。
- 提供 `sw.js` 以满足 PWA 安装条件。
- Service Worker 不缓存资源，只做全透传。

### 5.16 API 文档

项目提供 OpenAPI JSON 和文档页面：

- `/openapi.json`
- `/vip/openapi.json`
- `/api`
- `/vip/api`

## 6. API 规格

### 6.1 `GET /api/search`

搜索歌曲或专辑曲目。

查询参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `name` | 是 | 搜索关键词 |
| `source` | 否 | 音源，默认 `netease` |
| `pages` | 否 | 页码，默认 `1` |
| `count` | 否 | 每页数量，默认 `20` |
| `search_type` | 否 | `song` 或 `album`，默认 `song` |
| `vip` | 否 | VIP 模式标识，`1` 表示开启 |

成功响应：歌曲对象数组。

错误响应：

```json
{"error":"Keyword is required"}
```

### 6.2 `GET /api/cover`

代理返回封面图片。

查询参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `source` | 是 | 音源 |
| `id` | 是 | 封面 ID 或歌曲 ID |
| `vip` | 否 | VIP 模式标识 |

成功响应：图片二进制内容。

失败响应：404 空响应。

### 6.3 `GET /api/preview`

获取试听链接。

查询参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `source` | 是 | 音源 |
| `id` | 是 | 歌曲 ID |
| `vip` | 否 | VIP 模式标识 |

成功响应：

```json
{"url":"https://example.com/audio.mp3"}
```

### 6.4 `GET /api/info`

获取文件大小和音质等级。

查询参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `source` | 是 | 音源 |
| `id` | 是 | 歌曲 ID |
| `br` | 否 | 音质，默认 `999` |
| `vip` | 否 | VIP 模式标识 |

成功响应：

```json
{"size":"10.20MB","level":"无损音质"}
```

### 6.5 `POST /api/lyric`

仅下载歌词。

请求体：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `source` | 是 | 音源 |
| `id` | 是 | 歌曲 ID |
| `name` | 是 | 歌曲名 |
| `artist` | 否 | 歌手 |
| `vip` | 否 | VIP 模式标识 |
| `subdir` | 否 | 保存子目录 |

成功响应：

```json
{"status":"success","filename":"歌手 - 歌名.lrc","path":"/music/歌手/歌手 - 歌名.lrc"}
```

### 6.6 `POST /api/download/start`

创建后台下载任务。

请求体：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `source` | 是 | 音源 |
| `id` | 是 | 歌曲 ID |
| `name` | 是 | 歌曲名 |
| `artist` | 否 | 歌手 |
| `album` | 否 | 专辑 |
| `pic_id` | 否 | 封面 ID |
| `br` | 否 | 音质 |
| `lyric` | 否 | 是否同时写入歌词 |
| `vip` | 否 | VIP 模式标识 |
| `subdir` | 否 | 保存子目录 |

成功响应：

```json
{"job_id":"..."}
```

### 6.7 `GET /api/download/progress/{job_id}`

查询下载任务进度。

成功响应：

```json
{
  "status":"downloading",
  "progress":50,
  "filename":"",
  "error":"",
  "result":null
}
```

状态值：

- `queued`
- `downloading`
- `tagging`
- `success`
- `error`

### 6.8 `POST /api/download`

同步下载接口，参数与 `/api/download/start` 类似。主要用于兼容直接调用场景。

### 6.9 `GET /api/workflow`

解析分享链接并下载。

查询参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `text` | 是 | 分享链接或包含链接的文本 |
| `br` | 否 | 音质，默认 `999` |
| `vip` | 否 | VIP 模式标识 |
| `subdir` | 否 | 保存子目录 |

## 7. 数据与文件规则

### 7.1 下载根目录

下载根目录由环境变量 `DOWNLOAD_DIR` 指定，默认值为 `/music`。

支持 `~/` 路径展开。

### 7.2 文件命名

歌曲文件名：

```text
歌手 - 歌名.{ext}
```

歌词文件名：

```text
歌手 - 歌名.lrc
```

文件名中的以下字符会被移除：

```text
\ / : * ? " < > |
```

### 7.3 音频标签

支持写入标签的格式：

- MP3：ID3。
- FLAC：Vorbis Comment。
- M4A：MP4 atoms。

写入字段：

- 标题。
- 歌手。
- 专辑。
- 封面。
- 歌词。

## 8. 非功能需求

### 8.1 安全性

- 所有保存子目录必须限制在下载根目录内。
- 不允许绝对路径、空字符、`..` 路径片段。
- 前端展示第三方返回文本时必须转义 HTML。
- 下载目录、文件名、接口参数不应信任前端输入。

### 8.2 稳定性

- 第三方接口异常时应返回结构化错误信息。
- 下载任务应支持失败状态展示。
- 过期下载任务应被清理，避免内存无限增长。

### 8.3 性能

- 下载任务使用线程池后台执行。
- 前端通过轮询更新单曲下载进度。
- 封面图片代理响应可设置短期缓存头。
- 网易云音乐 2 单曲信息使用进程内 LRU 缓存降低重复请求。

### 8.4 可用性

- 移动端和桌面端都应可用。
- 页面刷新后保留常用选项。
- 下载过程应有明确进度反馈。
- 批量下载应显示总体成功和失败数量。

## 9. 验收标准

### 9.1 搜索

- 输入歌手或歌名并选择“歌手/歌曲”后，可以返回歌曲结果。
- 输入专辑名并选择“专辑”后，可以返回专辑曲目结果。
- 点击歌手名会切换为“歌手/歌曲”搜索。
- 点击专辑名会切换为“专辑”搜索。
- 翻页时当前播放停止，已选歌曲清空。

### 9.2 缓存

- 刷新页面后，搜索类型、搜索内容、音源、音质、保存目录恢复为上次值。
- 普通模式和 VIP 模式中不存在的缓存音源不会被强制恢复。
- 搜索类型变化时 placeholder 立即更新。

### 9.3 下载

- 单曲下载能创建任务并显示进度。
- 下载完成后文件保存到正确目录。
- 保存目录选择“歌手”时，文件进入歌手目录。
- 保存目录选择“专辑”时，文件进入专辑目录。
- 仅下载歌词遵循同样的保存目录规则。
- 批量下载能逐首执行，并在结束后显示成功/失败数量。

### 9.4 试听与信息

- 点击试听能播放音频。
- 切换歌曲试听会停止上一首。
- 点击信息按钮能展示音质或大小信息。

### 9.5 PWA

- 页面可被浏览器识别为可安装 PWA。
- Service Worker 不缓存接口或资源响应。

## 10. 未来可选增强

- 支持自定义保存目录模板。
- 快速链接下载支持按歌手或专辑目录保存。
- 搜索结果支持排序或过滤。
- 批量下载支持并发数配置。
- 增加下载历史记录。
- 增加失败重试队列。
